# 产品需求文档 (PRD)：基于感知机的中英文词性标注系统

## 1. 项目概述
[cite_start]本项目的目标是设计并实现一个能够处理中文和英文的自动词性标注（POS Tagging）系统 [cite: 1, 5][cite_start]。系统采用**结构化平均感知机（Structured Averaged Perceptron）**算法作为核心，配合 **Viterbi 解码**实现全局最优序列预测 [cite: 2, 3]。

## 2. 核心功能需求

### 2.1 数据处理模块
* [cite_start]**格式支持**：支持标准序列标注格式（如 CoNLL 格式）的加载与解析 [cite: 3]。
* [cite_start]**预处理**：实现低频词归一化、标签规范化以及数据集（训练/验证/测试）的 8:1:1 随机划分 [cite: 3]。
* **语料适配**：兼容中文（分词后的文本）与英文语料。

### 2.1.1 语料格式统一转换模块（新增）
本模块负责将中英文原始语料从各自的原始格式统一转换为标准 CoNLL 格式，作为整个数据流水线的第一步。

* **中文语料转换**：
    * 支持输入格式：PKU/MSR 格式（`词/词性` 空格分隔，如 `中国/NR 政府/NN`）。
    * 输出格式：标准 CoNLL 格式（每行一个词，字段含 `ID`、`FORM`、`UPOS` 等）。
    * 句子边界：以空行或换行符划分句子。

* **英文语料转换**：
    * 支持输入格式：NLTK/WSJ 格式（`word/TAG` 空格分隔，如 `The/DT cat/NN`）及 Penn Treebank（PTB）括号格式。
    * 输出格式：标准 CoNLL 格式，词性标签保持原始标签集（Penn Treebank Tags）。

* **通用处理逻辑**：
    * 自动检测输入语言（中文 / 英文）及格式类型。
    * 过滤无效行、空行、注释行。
    * 输出文件编码统一为 UTF-8。

* **交付形式**：
    * 独立脚本 `data_converter.py`：支持命令行调用，手动一次性转换原始语料。
    * 集成接口：在 `utils.py` 中暴露 `convert_to_conll()` 函数，供训练流程自动调用。

### 2.2 特征工程模块 (Feature Engineering)
[cite_start]这是模型性能的关键，需实现以下维度的特征提取 [cite: 2, 3]：
* **上下文特征**：当前词 $w_i$、前一个词 $w_{i-1}$、后一个词 $w_{i+1}$。
* **英文形态特征**：前缀/后缀（长度 1-3）、是否首字母大写、是否包含数字。
* **转移特征**：前一位置标签 $y_{i-1}$ 到当前标签 $y_i$ 的转移。

### 2.3 模型训练与推理模块
* **算法核心**：实现结构化感知机，使用线性打分函数进行序列评分：
    $$S(x, y) = \sum_{i=1}^{n} w \cdot \Phi(x, y, i)$$
* [cite_start]**训练策略**：引入**平均权重机制（Averaged Weights）**，通过累积权重的平均值来提升模型的收敛稳定性和泛化能力 [cite: 2, 3]。
* [cite_start]**全局解码**：集成 **Viterbi 算法**，在推理阶段寻找全局概率最高的标签序列 [cite: 3]。
* [cite_start]**基线对比**：实现至少一种对比方法（如 HMM 或最频词性标注）以验证模型效果 [cite: 2, 3, 5]。

### 2.4 Web 交互与可视化
* [cite_start]**后端 API**：基于 Flask 或 FastAPI 搭建，提供模型持久化加载与在线推理接口 [cite: 3, 5]。
* **前端界面**：
    * [cite_start]**文本输入区**：支持用户直接输入待标注的中英文句子 [cite: 5]。
    * [cite_start]**结果展示区**：以表格或带有彩色标签（Tag）的形式展示词性标注结果 [cite: 3]。
    * [cite_start]**数据可视化**：通过图表展示句子中的词性分布情况 [cite: 3]。

## 4. 关键指标 (KPI)
* [cite_start]**准确率 (Accuracy)**：在中英文标准语料库上达到稳定性能（参考目标：92% 以上） [cite: 2]。
* [cite_start]**鲁棒性**：能够处理未登录词（OOV），利用形态特征和上下文特征进行推断 [cite: 2]。
* [cite_start]**系统响应**：Web 界面推理延迟需控制在毫秒级 [cite: 1]。
